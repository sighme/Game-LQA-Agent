# Game-LQA-Agent
A multi-agent workflow for automated game localization and LQA based on LLMs.

项目名称： 基于多 Agent 协作的独立游戏本地化自动化 LQA（语言质量保证）流水线
1. 解决的核心痛点：
独立游戏文本量巨大（常达数十万词），且包含复杂的 JSON/XML 代码标签。传统 CAT 工具（如 memoQ）的内置 QA 只能进行死板的规则检查（如漏空格、缺标签），无法识别“机翻腔”、“死译”、语气不符以及结合游戏世界观（Lore）的上下文语境错误。而人工全量审校成本极高，导致独立游戏汉化质量难以把控。
2. 核心逻辑流与 Agent 协作：
我构建了一个基于脚本解析与大模型协作的自动化审校 Agent 工作流：
解析与预处理 Agent： 自动提取游戏底层 cutscene.json 等文件中的源语言与目标语言键值对，过滤掉非文本代码，将其结构化。
长链推理与语义审查 Agent（计划深度接入 MiMo）： 载入游戏的角色设定与专有名词表（Glossary）作为长上下文 System Prompt。Agent 需对比中英双语，进行逻辑推理，判断译文是否达到“信达雅”，并识别生硬的翻译腔。
标签校验 Agent： 交叉验证大模型的修改建议，确保换行符（如 \n）和变量参数未被破坏，防止游戏运行时崩溃。
闭环输出： 最终自动生成包含 [原文] | [当前译文] | [错误类型] | [修改建议] 的标准化双语 Excel 报告，便于我直接导回 memoQ 进行快速修正。
3. 目前成果与需求：
目前该工作流已在我个人的独立游戏汉化项目中跑通，将原本需要 2 周的人工审校流程缩短至 2 天，语义错误拦截率提升了 80% 以上。由于全量审校极其消耗 Token（每次需输入大量上下文），因此希望申请 MiMo 模型的 Token 额度，借助 MiMo 优秀的中文理解能力，进一步降低这套自动化本地化工作流的运行成本。
